Non polymorphic

In this chapter you’ll get a persister per child class, hence you can’t load all children entities of a mapped super class at once : for that, see polymorphism.

Stalactite supports two types of non-polymorphic inheritance :

  • mapped super class, that only shares mapped properties, not id policy
  • inheritance aimed at defining id policy

Mapped super class

To avoid describing several times the same persistence mapping of a common parent class, you can share it through an “embeddable” mapping and reference it in children classes mapping thanks to the mapSuperClass(..) method. Since this way of doing is just for skipping repetitive common mapping, it is not possible to define the target table of super class as well as its persistence policy : both are only done by children classes.

As an example, we can have a Vehicleclass inherited by Carand Truck: Cars will be stored in the Cartable with all its data, and Trucks will have its own stored in the Trucktable, independently. Such a mapping can be defined as this :

FluentEmbeddableMappingBuilder<Vehicle> vehiclePersistenceConfigurer =
    MappingEase.embeddableBuilder(Vehicle.class)
	.map(Vehicle::getColor);

MappingEase.entityBuilder(Car.class, Long.class)
	// concrete class defines id
	.mapKey(AbstractVehicle::getId, IdentifierPolicy.databaseAutoIncrement())
	.map(Car::getModel)
	.mapSuperClass(vehiclePersistenceConfigurer)
	.build(persistenceContext);
	
MappingEase.entityBuilder(Truck.class, Long.class)
	// concrete class defines id
	.mapKey(Truck::getId, IdentifierPolicy.databaseAutoIncrement())
	.map(Truck::getEngine)
	.mapSuperClass(vehiclePersistenceConfigurer)
	.build(persistenceContext);

Inheritance to define identifier policy

As a difference with embeddable super class, hereafter the shared parent mapping defines id policy. This inheritance is also declared by the mapSuperClass(..)method, but by passing it a entity configuration.

Here is a simple example where a configuration defines mapped properties of a Vehicle class, as well as its identifier policy, then the Car class can benefit of it.

EntityMappingConfiguration<Vehicle, Long> inheritanceConfiguration = MappingEase
      .entityBuilder(Vehicle.class, Long.class)
      // mapped super class defines id
      .mapKey(AbstractVehicle::getId, IdentifierPolicy.databaseAutoIncrement())
      .map(Vehicle::getColor)
      .getConfiguration();

EntityPersister<Car, Long> carPersister = MappingEase
      .entityBuilder(Car.class, Long.class)
      .map(Car::getModel)
      .mapSuperClass(inheritanceConfiguration)
      .build(persistenceContext);

By default, mapped properties in the parent configuration are stored in the child class table, but you may want to store them in different tables through a join on the primary key. For such a case one can simply ask for withJoinedTables(..) right after calling mapSuperClass(..). The method has two signatures, one without argument, meaning that the parent table will be named according to the configured table naming strategy, and one of them taking the table on which parent class persistence must be done. Here is the same example as the previous one with this option activated :

EntityMappingConfiguration<Vehicle, Long> inheritanceConfiguration = MappingEase
      .entityBuilder(Vehicle.class, Long.class)
      // mapped super class defines id
      .mapKey(AbstractVehicle::getId, IdentifierPolicy.databaseAutoIncrement())
      .map(Vehicle::getColor)
      .getConfiguration();

EntityPersister<Car, Long> carPersister = MappingEase
      .entityBuilder(Car.class, Long.class)
      .map(Car::getModel)
      .mapSuperClass(inheritanceConfiguration)
            .withJoinedTable(new Table("Vehicle"))
      .build(persistenceContext);
