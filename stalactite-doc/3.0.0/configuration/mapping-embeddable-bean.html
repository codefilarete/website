Mapping embeddable bean

Embedding beans follows the same principle as properties or enums, they are declarable thanks to the embed(..) method.

Supposing that a Country has a Timestamp which itself as a modificationDate and a creationDate that you want to persist into the same database table as the Country entity, then you have to embed the Timestamp (complex type) into the Country entity through the following way :

FluentMappings.entityBuilder(Country.class, Long.class)
    .embed(Country::getTimestamp, FluentMappings.embeddableBuilder(Timestamp.class)
		.map(Timestamp::getCreationDate)
		.map(Timestamp::getModificationDate))

Overriding properties column names

Sometimes embedded bean property names don’t match column ones, this requires to indicate column name that the property must fall into. This can be done by using the method overrideName(..) after declaring the embedded property :

FluentMappings.entityBuilder(Country.class, Long.class)
    .embed(Country::getTimestamp, FluentMappings.embeddableBuilder(Timestamp.class)
		.map(Timestamp::getCreationDate)
		.map(Timestamp::getModificationDate))
        .overrideName(Timestamp::getCreationDate, "creation_date")

Excluding properties from embedding

By default, all the defined target bean properties (please be aware of the note above: word “properties” may be misused) are candidates for being persisted. However, sometimes, it is necessary to remove some of them from the persistence. In such case, you could use the exclude(..) method :

FluentMappings.entityBuilder(Country.class, Long.class)
    .embed(Country::getTimestamp, FluentMappings.embeddableBuilder(Timestamp.class)
		.map(Timestamp::getCreationDate)
		.map(Timestamp::getModificationDate))
        .exclude(Timestamp::getCreationDate)

Embedding of embedding

In some cases, one may need to embed a bean of an already-embedded bean, then you should use the embed(..) method as in this (totally fake) example where president modification date is used as the election date stored in the country table :

FluentMappings.entityBuilder(Country.class, Long.class)
    .embed(Country::getPresident, FluentMappings.embeddableBuilder(Person.class)
		.map(Person::getId)
		.map(Person::getName)
		.embed(Person::getTimestamp, FluentMappings.embeddableBuilder(Timestamp.class)
			.map(Timestamp::getCreationDate)
			.map(Timestamp::getModificationDate))
			.overrideName(Timestamp::getModificationDate, "presidentElectedAt"))
