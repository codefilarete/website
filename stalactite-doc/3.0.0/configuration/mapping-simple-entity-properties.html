Mapping simple entity properties

From the orm module, the entry point is FluentMappings. The first step is to give class entity and its identifier type to the entityBuilder(..) method, such as this :

FluentMappings.entityBuilder(Country.class, Long.class)

Then you can add simple properties by chaining them with the map(..) method

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName)

Note that setters can be used as well :

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::setName)

If your field name doesn't match your accessor (your object doesn't follow the Java Bean Naming convention), then you can specify it :

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::setName).fieldName("countryName")

If the property is not accessible from outside by any getter or setter, you can target it by specifying its name. This can be useful for internally computed properties :

FluentMappings.entityBuilder(Country.class, Long.class)
    .map("classifiedUfoFilePassword")

Identifier

Defining which property identifies your entity is done with the mapKey(..) method. An identifier management policy must be given as argument.

FluentMappings.entityBuilder(Country.class, Long.class)
    .mapKey(Country::getId, IdentifierPolicy.afterInsert())
    .map(Country::getName)

See the Identifier Policy chapter to see the available identifier policy.

Composite identifier

Similarly to a single column identifier, you can define a composite-key with the mapCompositeKey(..) method. The difference with a single-column key is that you shall detail the mapping of your composite-object as a second argument of the method. Supposing you have Person and its identifier is a PersonId, the definition can be done as such :

FluentMappings.entityBuilder(Person.class, PersonId.class)
    .mapCompositeKey(Person::getId, FluentMappings.compositeKeyBuilder(PersonId.class)
        .map(PersonId::getFirstName)
        .map(PersonId::getLastName)
        .map(PersonId::getAddress))
    .map(Person::getAge)

Please note that a composite-key must always be assigned before insert. Hence, behind the scene, the identifier policy is already-assigned.

Additional options

Declaring property as mandatory

By default, Object properties are considered nullable, if that's not the case for your property, you can call the mandatory() method.

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).mandatory()

This will add a non-null constraint on the database column, which will throw an exception in case of null value at insertion and update time.

Conversely, by default, primitive type properties are considered mandatory; thus it is not necessary to declare them as such.

However, you can overcome this behavior by calling the nullable() method on the property mapping.

Declaring property as read-only

If you don't want Stalactite to write your property to the database, then you can mark it with the readonly() method on the property mapping.

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).readonly()

Converting value

You may need to adapt the value coming from the database. To make it, you should use the readConverter(..) method to specify a Converter. Here is an example that upper cases a name :

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).<String>readConverter(String::toUpperCase)

The call to readConverter(..) might be prefixed by the generic type of the source column to fix compilation issues.

You do the same for writing to the database with the writeConverter(..) method and specify a Converter too. Here is an example that trims a name :

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).writeConverter(String::trim)

A more advanced feature may be required if your property type doesn't match column one (considering you map your entities onto an existing database schema that you can hardly change). For example, if you have to handle some date written as text in the database, but you want to manage Date on the Java side (prefer the new java.time API instead of java.util.Date of course, this is only demo purpose). To fulfill this goal, you shall use the sqlBinder(..) method and specify a ParameterBinder ParameterBinder

SimpleDateFormat dateFormatter = new SimpleDateFormat("yyyy-MM-dd");
    FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getCreationDate)
    .sqlBinder(new NullAwareParameterBinder<>(
            // converting String to a Date when reading from the database
            DefaultParameterBinders.STRING_BINDER.thenApply(dateFormatter::parse),
            // writing the Date as a String when writing to the database
            DefaultParameterBinders.STRING_BINDER.preApply(dateFormatter::format)))

Setting a column name

You can override a property column name by adding it as an argument of the map(..) method.

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).columnName("isoCode")

You can also provide the column size by chaining your call with columnSize(..) and passing it a Size instance. The Size class provides factory methods to produce them for character and numeric types. 

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).columnSize(Size.length(50))

Setting a column

If you already have a Table instance (a Stalactite orm class) representing the database table and its Column instances you can reference them in the map(..) method, as you do for a column name.

Column type and property type must match, this is guaranteed by generics type of the map(..) method signature.

// Declaring a Table representing the one of the database schema
Table countryTable = new Table("country");
Column<Table, String> isoCodeColumn = countryTable.addColumn("isoCode", String.class);

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).column(isoCodeColumn)

Adding unique constraint

To add a unique constraint on a property, you should chain your declaration with unique().

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).unique()

Storing in a separate table

If you need your property to be stored in some extra table, then you should chain your declaration with an extraTableName(..) call.

FluentMappings.entityBuilder(Country.class, Long.class)
    .map(Country::getName).extraTableName("SecondaryCountryTable")
