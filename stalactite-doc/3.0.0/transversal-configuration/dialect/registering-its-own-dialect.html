Registering its own Dialect

Dialects are discovered thanks to the Java Service Provider system: Stalactite will look for subclasses of DialectResolverEntry in the classpath. Hence, some files named org.codefilarete.stalactite.sql.DialectResolver$DialectResolverEntry must be present in it and should contain a reference to the implementation of DialectResolverEntry: this interface contains some methods that work closely:

  • getDialect() is expected to return an instance of the Dialect to use
  • getCompatibility() should return a DatabaseSignet that describes the database (name and version) with which the Dialect is compatible
  • getVendorSettings() should return some DatabaseVendorSettings which are the default vendor ones
