Naming strategies

Table naming strategy

You may need to give a different policy than the default one that takes the mapped class name as the table name. To do it, you should call tableNamingStrategy(..) when building your mapping and pass it an implementation of TableNamingStrategy.

The default table naming strategy is TableNamingStrategy.DEFAULT

Column naming strategy

The default column naming strategy uses the mapped property name as the one for the column. To change it, one should use columnNamingStrategy(..) when building your mapping, by passing it an implementation of ColumnNamingStrategy.

The default column naming strategy is ColumnNamingStrategy.DEFAULT

Foreign key and join column naming strategy

Foreign keys and join columns can be named differently than the default strategy by applying some new ForeignKeyNamingStrategy and ColumnNamingStrategy during mapping with the respective foreignKeyNamingStrategy(..) and joinColumnNamingStrategy(..) methods.

  • The default foreign key naming strategy is ForeignKeyNamingStrategy.DEFAULT
  • The default join column naming strategy is ColumnNamingStrategy.JOIN_DEFAULT
  • ForeignKeyNamingStrategy.HASH and ForeignKeyNamingStrategy.HIBERNATE are also available

Association table naming strategy

When mapping List association, you may need to change association table naming strategy, then use the method associationTableNamingStrategy(..) with an instance of AssociationTableNamingStrategy as argument.

The default association table naming strategy is AssociationTableNamingStrategy.DEFAULT
